# Workflow Git simple

## Règle

Un commit = une petite étape fonctionnelle.

## Workflow

1. Choisir un ticket.
2. Lire les critères de validation.
3. Faire la modification dans Godot.
4. Tester.
5. Corriger les erreurs.
6. Faire un commit.
7. Passer au ticket suivant.

## Messages de commit recommandés

```text
Initial Godot project
Add main scene
Add static platform
Add tank scene
Add tank visual
Move tank left and right
Add fixed camera
Add turret aiming
Add shot charge UI
Charge shot with space
Add projectile scene
Fire projectile on space release
Add ballistic projectile motion
Add target scene
Detect target hit
Update documentation for demo
```

## Branches

Pour ce prototype, rester sur une seule branche : `main`.
